## What Gets Collected

When a valid email is submitted, the app sends:

- `email` (required for submission)
- `ens` (optional, normalized to lowercase)
- `alias`
- `source` (`alias` or `end_screen`)
- `weeks`
- `archetype`
- `ts` (ISO timestamp)

It also stores locally in browser `localStorage`:

- `internoun_email`
- `internoun_ens`

## Supabase Setup

### 1) Get API credentials

From your Supabase project:

- **Project URL**
- **anonymous API key** (safe for frontend; do NOT use service role key)

### 2) Create table

Run this in Supabase SQL Editor:

```sql
create table if not exists public.email_captures (
  id bigint generated by default as identity primary key,
  email text not null,
  ens text,
  alias text,
  source text,
  weeks integer,
  archetype text,
  ts timestamptz default now()
);
```

### 3) Enable RLS + insert policy

```sql
alter table public.email_captures enable row level security;

create policy "allow anon inserts to email_captures"
on public.email_captures
for insert
to anon
with check (true);
```

> Optional hardening: add rate limits with Edge Functions or a server proxy.

## App Configuration

- `SUPABASE_URL`

- `SUPABASE_TABLE`

Replace placeholders with your real values.

## Environment Variables

This repo includes `.env.example` for standardization.

Because this is a plain HTML file, `.env` is **not automatically loaded** by the browser.
Use one of these approaches:

1. Build tool (Vite/Next/Webpack) and inject env vars at build time
2. A pre-deploy replacement script
3. Move submit logic to a backend endpoint (recommended for stronger security)

## Security Notes

- Never expose `service_role` or secret keys in frontend code.
- `anonymous API key` is expected to be public.
- Protect your data with RLS policies and validation.

## Local Run

Open `internoun-dao-deck (12).html` in a browser, or serve the folder:

```bash
python3 -m http.server 8080
```

Then visit `http://localhost:8080`.
